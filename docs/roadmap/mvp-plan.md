# MVP Plan

The Minimum Viable Product (MVP) for Maxedu's library module focused on the core borrowing lifecycle.

## 1. Core Lifecycle
- **Unified Catalog**: A searchable list of all library books.
- **Request & Approval**: Basic workflow for members to request books and staff to approve them.
- **Issue & Return**: Functional endpoints for staff to record book issuance and returns.

## 2. Basic Member Portals
- **Student Portal**: View history and current borrowings.
- **Staff Portal**: Approve requests and process transactions.

## 3. Mandatory Infrastructure
- **Frappe/Nuxt Integration**: Secure communication between the backend and frontend.
- **Role-Based Access**: Primary roles (Student, Staff) with basic permission sets.

The MVP ensured that the library could transition from manual to digital operations with minimal friction.
